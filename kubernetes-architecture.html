⎈ Core Concepts
  • Control Plane manages cluster state via etcd. API Server is the single entry point
  • Worker Nodes run the actual workloads. Kubelet is the local agent on each node
  • Pods are the smallest deployable unit. Co-located containers share network namespace
  • Controllers continuously reconcile desired state vs actual state
  • Labels & Selectors connect Services → Pods. Annotations store metadata
  • Namespaces provide logical isolation with ResourceQuota enforcement
🕸️ Service Mesh Benefits
  • Zero-config mTLS: automatic mutual TLS between all services
  • Traffic management: canary, A/B testing, blue/green via VirtualService
  • Circuit breaking: automatic outlier detection via DestinationRule
  • Observability: L7 metrics, traces, logs without app changes
  • Sidecar pattern: Envoy proxy injected automatically via MutatingWebhook
  • Service identity: SPIFFE/SPIRE certificates for workload identity
💽 Storage Best Practices
  • Use StorageClass with WaitForFirstConsumer for zone-aware provisioning
  • NFS for ReadWriteMany (RWX) — shared across pods and nodes
  • EBS/GCE PD for ReadWriteOnce (RWO) — single node high performance
  • StatefulSet with VolumeClaimTemplates for per-pod persistent volumes
  • Encrypt Secrets at rest using KMS — base64 is NOT encryption
  • Use External Secrets Operator to sync from Vault/AWS Secrets Manager
🌐 Networking Flow
  • Each Pod gets unique IP via CNI plugin (Calico, Flannel, Cilium)
  • Services get stable ClusterIP via kube-proxy iptables/IPVS rules
  • Ingress controller routes L7 HTTP(S) traffic to Services
  • NetworkPolicy enforces microsegmentation at L3/L4 layer
  • CoreDNS resolves: my-svc.my-ns.svc.cluster.local
  • Cilium eBPF replaces kube-proxy for high-performance networking
📋 Workload Patterns
  • Deployment: stateless apps with rolling updates and rollback
  • StatefulSet: databases, message queues with stable network identity
  • DaemonSet: one pod per node for logging, monitoring, CNI agents
  • Job/CronJob: batch processing, scheduled tasks, data migrations
  • HPA scales replicas on CPU/memory; KEDA on custom event sources
  • Init containers run sequentially before main containers start
🔒 Security Layers
  • RBAC: Role/ClusterRole + Binding controls API access
  • PodSecurity: Restricted/Baseline/Privileged admission standards
  • NetworkPolicy: default-deny ingress/egress with explicit allows
  • OPA Gatekeeper: policy as code with ConstraintTemplates
  • Service mesh mTLS: encrypted east-west traffic with identity
  • Secrets rotation: External Secrets Operator + Vault dynamic secrets
