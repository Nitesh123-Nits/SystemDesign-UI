2
DNS Cache Lookup

Check Local Caches

● waiting

Before querying any DNS server, the system checks several layers of cache to avoid slow network lookups. This is the fastest possible resolution.

Cache lookup order: ① Browser DNS cache ← fastest, in-memory ② OS DNS cache ← system-level ③ /etc/hosts file ← static overrides ④ Router/LAN cache ← local network // Each entry has a TTL (Time To Live) TTL expired → proceed to DNS query Cache hit → skip DNS entirely ⚡

5
Network

TCP Three-Way Handshake

● waiting

Now we have an IP! The browser opens a TCP connection to port 443 (HTTPS). TCP requires a three-step handshake before any data flows, ensuring both sides are ready.

SYN Browser → Server "Can we talk?" SYN-ACK Server → Browser "Yes! Ready." ACK Browser → Server "Great, starting now." // RTT (round trip time) adds latency here Typical latency: 1–100ms depending on distance
6
Security

TLS Handshake (HTTPS)

● waiting

For HTTPS, a TLS handshake happens on top of TCP. The server presents its certificate, the browser verifies it with a Certificate Authority, and they negotiate an ephemeral session key for encrypted communication.

TLS 1.3 Handshake: ClientHello → ciphers, TLS version, random nonce ServerHello → chosen cipher, certificate Cert verify → browser checks CA signature Key exchange → ECDHE → shared secret derived Finished → 🔒 encrypted channel open // TLS 1.3 does this in 1-RTT (vs 2-RTT in TLS 1.2)
7
HTTP

HTTP Request & Response

● waiting

The browser sends an HTTP GET request over the encrypted tunnel. The server processes it and responds with headers and the HTML body. Modern connections use HTTP/2 multiplexing.

REQUEST → GET / HTTP/2 Host: www.example.com Accept: text/html Accept-Encoding: gzip, br RESPONSE ← HTTP/2 200 OK Content-Type: text/html; charset=utf-8 Content-Encoding: br Cache-Control: max-age=3600
8
Browser Engine

Render the Page

● waiting

The browser's engine parses HTML into a DOM tree, CSS into a CSSOM, combines them into a Render Tree, runs Layout, then Paint. JavaScript may block or defer this pipeline.

Critical Rendering Path: HTML → DOM (parse tags, build tree) CSS → CSSOM (parse styles) DOM + CSSOM → Render Tree Render Tree → Layout (compute positions) Layout → Paint (pixels on screen) Paint → Composite (GPU layers) // First Contentful Paint (FCP) ← user sees something
